📚 Lecture: The DP Approach

The Change-Making Problem seeks the absolute minimum coins to sum to W. This is a classic optimization problem.

Optimal Substructure: The best solution for an amount i is built using the best solutions already found for smaller amounts.

dp[i] = min(dp[i], dp[i - coin] + 1)

The DP Philosophy

  • Tabulation: We solve for $1, $2, $3... in order. By "memoizing" (saving) these in a table, we never calculate the same amount twice.
  • The Greedy Trap: A Greedy algorithm picks the largest coin first (a local choice). This often misses the Global Optimum (the absolute best total solution) for non-standard coins.
  • Global Optimum: The single best answer that exists. DP guarantees to find it by exploring every possible combination.
